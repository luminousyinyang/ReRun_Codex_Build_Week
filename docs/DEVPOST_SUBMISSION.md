# Devpost submission guide

## Category and positioning

Select **Education**. Use this concise, human-edited framing:

> ReRun is an interactive retro-TV study app where the show cannot continue until the learner answers. It turns retrieval practice, corrective feedback, adaptive re-explanation, and review into TV mechanics: question beats, consequence scenes, rewinds, and commercials.

Describe only the features that are visibly working. Do not describe P1/P2 items as shipped, make unsupported learning-outcome claims, or paste unedited model-generated copy.

## Required submission materials

- Working project built with Codex and GPT-5.6.
- Selected category: Education.
- Project name, tagline, and edited description.
- Public YouTube demo video under three minutes; voiceover must explain what was built and how both Codex and GPT-5.6 were used.
- Repository URL. If private, share it with `testing@devpost.com` and `build-week-event@openai.com`.
- README with setup, demo/sample data, judge path, and Codex/GPT-5.6 explanation.
- `/feedback` Codex session ID for the session where most core work occurred.
- Submit, not Draft.

## Video beat sheet (under 3:00)

| Time | Show |
| --- | --- |
| 0:00-0:15 | CRT power-on and hook: “What if your study show needed you?” |
| 0:15-0:35 | Student problem and active-learning framing; avoid uncited efficacy promises. |
| 0:35-1:50 | One uninterrupted demo: load course, question beat, intentional wrong answer, consequence, rewind, correction, commercial. |
| 1:50-2:15 | Cliffhanger, due-time framing, and the zero-key judge experience. |
| 2:15-2:45 | Architecture diagram, schema contract, and concise explanation of Codex/GPT-5.6 roles. |
| 2:45-3:00 | Impact framing, hosted URL, and repository. |

## Compliance checklist

- [ ] The entry uses Codex and GPT-5.6; the usage is specific and truthful.
- [ ] The application is working/runnable, and judges can use the bundled demo without secrets.
- [ ] Repo is public with appropriate license, or private and shared with both required judge addresses.
- [ ] Video is public on YouTube, under three minutes, and has voiceover covering Codex and GPT-5.6.
- [ ] `/feedback` session ID is entered.
- [ ] Education is selected; project description is human-edited.
- [ ] All team members are correctly listed if applicable.
- [ ] Shipped visuals, characters, sound, music, and footage are original, licensed, or generated for ReRun; no third-party trademarks, characters, or copyrighted music appear.
- [ ] Final name receives human trademark/name review.
- [ ] Live rules, eligibility, deadlines, and submission fields are checked again immediately before submission.

## AI-review-friendly evidence

Make the project easy to inspect: clear README, deterministic fixture, typed contracts, actual tests, screenshots/video of the expected demo path, meaningful commits, and an honest `CODEX_LOG.md`. Avoid buzzwords, vague “AI-powered” claims, hidden setup steps, secret-dependent demos, or simulated features presented as live.
